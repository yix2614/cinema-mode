# 全屏模式布局网格规范 (Grid Spec)

当右侧评论面板 (Comment Panel) 打开时，页面采用动态响应式网格系统进行布局。

## 1. 布局逻辑原理
布局基于 **Grid 栏位 (Columns)** 计算，动态分配给视频区域和评论面板。
- **总宽度**: `100vw` (屏幕宽度)
- **可用宽度**: `100vw` - `2 * Margin`
- **栏位宽度 (Col Width)**: `(可用宽度 - (Total Cols - 1) * Gap) / Total Cols`

---

## 2. 响应式断点规则

**超大屏 (> 1600px)**
- 栅格总数: 24
- 间距 (Gap): 16px
- 页边距 (Margin): 16px
- 评论面板占用: 6 栏

**宽屏 (1201px - 1600px)**
- 栅格总数: 20
- 间距 (Gap): 16px
- 页边距 (Margin): 16px
- 评论面板占用: 6 栏

**标准屏 (840px - 1200px)**
- 栅格总数: 12
- 间距 (Gap): 8px
- 页边距 (Margin): 8px
- 评论面板占用: 4 栏

**小屏 (< 840px)**
- 栅格总数: 12
- 间距 (Gap): 8px
- 页边距 (Margin): 8px
- 评论面板占用: 4 栏

> **注意**: 虽然 Grid 定义了 Panel Cols，但我们有一个硬性的**最小宽度 (Min-Width) 限制为 288px**。
> - 在大屏上，Panel Cols 计算出的宽度通常远大于 288px，遵循 Grid 比例。
> - 在极小屏幕上（如 < 840px），如果 4 栏宽度计算结果小于 288px，**则强制使用 288px**，此时 Panel 不再严格对齐 Grid，而是优先保证内容可读性，视频区域会被相应压缩。

---

## 3. 评论面板尺寸计算
评论面板的宽度不是固定像素，而是基于它所占用的 **Panel Cols** 动态计算得出。

- **计算公式**:
  ```
  Panel Width = (Col Width * Panel Cols) + ((Panel Cols - 1) * Gap)
  ```
- **最小宽度限制**:
  无论 Grid 计算结果如何，Panel Width 始终 **>= 288px**。
  *(当屏幕极窄导致计算宽度小于 288px 时，强制使用 288px，此时视频区域会相应被挤压)*

---

## 4. 间距与留白 (Padding/Margin)

### 视频容器 (Video Container)
当评论面板打开时，视频容器会有额外的内边距来制造视觉呼吸感：

| 属性 | `Width >= 1201px` | `Width < 1201px` |
| :--- | :--- | :--- |
| **左侧 Margin** | **16px** | **8px** |
| **垂直 Margin** | 上下各 **16px** | 上下各 **8px** |

### 评论面板容器 (Comment Panel Wrapper)
评论面板周围的留白：

| 属性 | `Width >= 1201px` | `Width < 1201px` |
| :--- | :--- | :--- |
| **上/右/下 Padding** | **16px** | **8px** |
| **左侧间距** | 保持 **Gap** 间距 | 保持 **Gap** 间距 |
